# 05. Canva Standards(Canva 移植標準)

## 前提

- Canva コネクタ(MCP)は**未接続**が既定状態。接続にはユーザーの OAuth 承認が必要(承認境界)。
- 接続前提の作業を約束しない。未接続の間は「Canva ブリーフ(移植仕様書)」を成果物とする。

## Canva ブリーフとは

人間(または接続後の自動化)が Canva 上でデッキを再現するための仕様書。`templates/canva_brief_template.md` を使う。必須項目:

- デッキ名・目的・想定視聴者
- スライドごとの: レイアウト種別 / 見出しテキスト / 本文テキスト / 図版指示 / 配色指定(HEX)
- フォント指定(Canva 内で近いもの: 例 Yu Gothic → Noto Sans JP)
- 使用するCanva要素の種類(グリッド、カード、バッジ、チャート)
- アニメーション指定(使う場合のみ。既定は「なし」)

## デザイン規則

- PowerPoint 版と同一パレット・同一構成を維持する(ブランド一貫性)。
- Canva テンプレートをそのまま流用する場合はライセンス(無料/Pro素材)をブリーフに明記する。
- Pro 素材・有料素材の使用は課金に関わるため人間承認が必要。

## 接続後の自動化(将来)

Canva コネクタが承認・接続されたら:

1. ブリーフからデザインを自動生成するフローを Issue 化して提案する。
2. 生成結果の公開・共有リンク発行は人間承認の後に行う。
