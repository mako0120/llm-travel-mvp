# 01. Core Rules(絶対規則)

以下は他のすべてのドキュメントより優先される。

## 承認境界(人間承認なしに実行しない)

- `main` への直接 push / merge
- 本番公開・デプロイ・外部への発信(SNS投稿、動画公開、記事公開を含む)
- 課金・サブスクリプション契約・購入
- 新しい外部連携のインストール・OAuth 認可・権限付与
- 認証・データベース・環境変数・外部サービス設定の変更
- リポジトリの作成・削除・可視性変更

## 秘密情報

- API キー・トークン・`.env` 系ファイルを表示・保存・コミットしない。
- 秘密情報らしき文字列を発見したら、内容を出力せずに場所だけ報告する。

## 正直さ

- ツール・スキル・MCP・コネクタは**実際に検証してから**「利用可能」と言う。
- 分類は4値で記録する: `verified` / `usable with setup` / `unavailable` / `unsafe or unnecessary`。
- 検証できなかった品質チェック(例: ピクセル単位の見た目確認)は「できなかった」と明示する。

## 著作権

- 他者のスライド・画像・文章をコピーするワークフローを作らない。
- 図・チャート・レイアウトはオリジナルで生成する。
- 引用は出典明記の短い引用に限る。

## 利用規約の遵守

- 料金制限の回避、複数アカウントの併用、API の不正利用は行わない。
- 「無料化」「無制限化」をうたう非公式ツールは、公式ドキュメントで正当性を確認できない限り採用しない。

## 作業規律

- すべての変更は open な GitHub Issue から始める。
- ブランチは `claude/feature-<issue番号>-<slug>`。
- PR は Draft で作成し、本文に `Closes #<issue番号>` を書く。
- Issue のスコープ外を変更しない。無関係なリファクタリングをしない。
- 他のエージェント(chatgpt/, codex/)のブランチに上書きしない。
