# 06. Content R&D(コンテンツ調査・研究)

## 調査の原則

- 情報源の優先順位: 公式ドキュメント > 公式発表 > 一次リポジトリ > 論文 > 信頼できる技術ブログ > コミュニティ(発見用途のみ)。
- 取得できない情報を推測で埋めない。「不明」と記録する。
- すべての事実に URL 出典を付ける。出典のない数字は使わない。
- 記録は4分類: 検証済み事実 / 合理的推測 / 推奨 / 不明。

## 調査レポート

`templates/research_report_template.md` を使う。必須構成:

1. 調査目的と問い
2. 検証済み事実(出典付き)
3. 比較表(候補ごとに: 提供元 / 用途 / 導入条件 / ライセンス / 更新状況 / セキュリティ / 期待効果 / リスク / 判定)
4. 判定: adopt / trial / watch / reject
5. 不明点と追加調査計画

## コンテンツ研究の観点(動画・記事・プレゼン共通)

- タイトル: 数字・ベネフィット・具体性の有無
- 構成: 冒頭30秒(スライドなら最初の3枚)で課題提示できているか
- 維持: 1トピック1メッセージ、転換の頻度
- CTA: 次のアクションが1つに絞られているか
- 対象: 初心者向け/専門家向けの明示

## 失敗事例の収集

成功例と同じ重みで失敗例(例: 誤回答を放置したチャットボット導入失敗)を集め、
「同じ失敗をしないためのチェック項目」に変換して該当ドキュメントへ反映する。
