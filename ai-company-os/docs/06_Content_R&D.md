# 06. Content R&D(コンテンツ調査・研究)

## 調査の原則

- 情報源の優先順位: 公式ドキュメント > 公式発表 > 一次リポジトリ > 論文 > 信頼できる技術ブログ > コミュニティ(発見用途のみ)。
- 取得できない情報を推測で埋めない。「不明」と記録する。
- すべての事実に URL 出典を付ける。出典のない数字は使わない。
- 記録は4分類: 検証済み事実 / 合理的推測 / 推奨 / 不明。

## 調査レポート

`templates/research_report_template.md` を使う。必須構成:

1. 調査目的と問い
2. 検証済み事実(出典付き)
3. 比較表(候補ごとに: 提供元 / 用途 / 導入条件 / ライセンス / 更新状況 / セキュリティ / 期待効果 / リスク / 判定)
4. 判定: adopt / trial / watch / reject
5. 不明点と追加調査計画

## コンテンツ研究の観点(動画・記事・プレゼン共通)

- タイトル: 数字・ベネフィット・具体性の有無
- 構成: 冒頭30秒(スライドなら最初の3枚)で課題提示できているか
- 維持: 1トピック1メッセージ、転換の頻度
- CTA: 次のアクションが1つに絞られているか
- 対象: 初心者向け/専門家向けの明示

## テーマ選定モード(2026-07-21〜、オーナー指示によりデフォルト運用)

テーマ選定は2つのモードを使い分ける。指示がなければ**バイラル優先モード**をデフォルトとする。

### モードA: 評価表モード(従来)

`research_report_template.md` の9項目採点(日本需要・新しさ・参入余地・実用性・継続性・
差別化・スライド化・Shorts性・収益化、100点満点、70点未満は不採用)で候補を比較する。

### モードB: バイラル優先モード(デフォルト)

「再生数がとれている、または取れそうなコンテンツ」を優先して選ぶ。日本・海外どちらも対象。

- 採用条件: **実証済みのバイラル実績**があること。例: 実際にSNSでトレンド入りした
  (X公式トレンドページ等で確認できる)、複数の信頼できるメディアが「話題」「バズった」
  と報じている、公式発表の実績数値(会員数・売上急増等)が確認できる、など
- **却下条件**: SEO対策ブログ・AI動画生成ツールの宣伝記事が主張する再生数(「◯◯万回再生」
  等)は、一次ソース(実際の動画URL)が示されない限り採用しない。不自然に丸い数字・
  検証不能な数字は信頼性が低いと判断し除外する
- 出典の優先順位はモードAと同じ(公式発表・信頼できる報道 > SEOブログ)
- 対象ジャンルの配分(AI/エージェント系70%・海外/国内で活躍する日本人20%・
  急成長アカウント10%)は維持するが、各枠内でモードBの基準を優先する

## 失敗事例の収集

成功例と同じ重みで失敗例(例: 誤回答を放置したチャットボット導入失敗)を集め、
「同じ失敗をしないためのチェック項目」に変換して該当ドキュメントへ反映する。
