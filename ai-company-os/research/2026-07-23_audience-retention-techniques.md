# 調査結果: 視聴者が飽きない動画・プレゼン台本にするための技法

調査日: 2026年7月23日
調査目的: オーナー指示「30枚欠かさず、視聴者が飽きないようにネットで学べ」を受け、
PowerPoint(30枚)+AI対話ナレーションで視聴離脱を防ぐための具体的な技法を調べ、
`docs/04_PowerPoint.md` / `docs/13_AI_Dialogue_Script.md` の制作ルールに反映する。

## 検証済み事実(出典付き)

| 事実 | 出典 |
|---|---|
| 最初の30秒(特に冒頭3〜10秒)の設計が、その後の視聴継続を大きく左右する。挨拶や自己紹介から入ると離脱を招きやすい | [teleprompter.com](https://www.teleprompter.com/blog/youtube-audience-retention) / [manabi-choice.jp](https://manabi-choice.jp/retention-video-opening-rule/) / [techgym.jp](https://techgym.jp/column/doga-39/) |
| 90〜120秒に1回、「パターンインタラプト」(カメラ角度の変化、グラフィック、効果音、ナレーションペースの変化、画面上の問いかけ等、意図的な刺激の変化)を入れると離脱が減る | [teleprompter.com](https://www.teleprompter.com/blog/youtube-audience-retention) |
| ペースが遅すぎると視聴者は退屈して離脱し、速すぎると圧倒されて離脱する。ジャンプカット・ズームイン・画面上テキストで調整するのが有効 | 同上 |
| 話すだけでなく「見せる」ことが重要。概念の説明には図解、手順の説明には視覚的なリスト表示を伴わせると、トーキングヘッドのみの形式より視聴維持率が高い | 同上 |
| 語り手のエネルギーレベルは視聴維持率に直結する。一定のトーンではなく、意図的に緩急・強弱をつける(dynamic range)ことが有効 | 同上 |
| 「約束する→提供する→次を約束する」の反復構造(Promise → Deliver → New Promise)によって、視聴者は満足感と次への期待を交互に得られ、視聴を継続しやすくなる | 同上 |
| 台本作成時は、冒頭フックの候補を最低3案出して最も強いものを選ぶ手法が推奨されている | [techgym.jp](https://techgym.jp/column/doga-39/) |

## 合理的推測(事実と区別)

- (推測)これらの技法は主に単独ナレーション動画を前提にしたノウハウであり、2AIペルソナの
  対話形式にそのまま適用できるかは検証されていないが、対話形式は本質的に「話者の切り替え」
  自体がパターンインタラプトになりうるため、相性は良いと考えられる。

## 不明点

- 日本語ナレーション・スライド形式のコンテンツにおける、これらの技法の効果測定データ
  (今回参照した情報源はいずれもYouTube動画全般を対象としており、スライド解説動画・
  対話形式コンテンツに特化した定量データではない)

## 制作ルールへの反映方針

上記を踏まえ、以下の形で `docs/04_PowerPoint.md`(スライド設計)と
`docs/13_AI_Dialogue_Script.md`(対話ナレーション設計)に反映する。

| 技法 | PowerPoint側の反映 | 対話ナレーション側の反映 |
|---|---|---|
| 冒頭フック | 表紙スライドで結論の一部・驚きの数字を匂わせる(挨拶のみで終わらせない) | 起の1行目を挨拶や自己紹介ではなく、フックになる問いかけ・驚きの一言にする |
| パターンインタラプト(90〜120秒毎) | 同一レイアウトを4連続以上使わない(`build_deck.py`で検証) | 同じトーン(事実→相槌)の反復を避け、驚き・ツッコミ・問い直しを混ぜる |
| 話すだけでなく見せる | 数字を出すスライドは必ずchart/table/cardsを使う(既存ルール継続) | 数字を話す行は、対応スライドの視覚要素(グラフ・表)を指す一言を添える |
| 約束→提供→次の約束 | agendaスライドで予告し、各章の最後に次章への一言を入れる | 各幕の最後に、次の幕へのフックとなる一言(オープンループ)を入れる |
| エネルギーの緩急 | (スライドでは直接表現しづらいため、ノート・台本側で対応) | ナビAI/リサーチAIのトーンに意図的な起伏をつける(驚き・共感・ツッコミ等) |
