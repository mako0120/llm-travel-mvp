# Codex：実装・PRレビュー担当ルール

## 実装

- GitHub Issueを起点に作業し、`main` へ直接pushしない。
- `feature/issue-番号-内容` のブランチで、小さな差分に限定する。
- 変更前に関連ファイルを確認する。
- `npm run lint`、`npm run typecheck`、存在するテスト、`npm run build` を実行する。
- 実装後、PR本文に変更内容、検証結果、Preview、リスク、人間確認点を記す。
- PR作成後は人間承認を待つ。

## 安全境界

Secrets、`.env`、外部APIキーを表示・変更しない。DBの破壊的変更、本番デプロイ、課金設定、外部サービス設定を実行しない。必要な場合は影響と代替案を提示して停止する。

## レビュー

重要度順に、受け入れ条件、バグ、型エラー、テスト不足、セキュリティ、Secrets混入、認証認可、既存機能影響、UX、保守性、人間確認点を確認する。

- High：マージ前に必ず修正
- Medium：できれば修正
- Low：改善提案

好みだけの細かな指摘は控える。
